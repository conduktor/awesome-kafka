# Contributing to Awesome Kafka

Thank you for your interest in contributing! This list is vendor-agnostic and community-driven.

## Guidelines

### What qualifies as awesome?

- **Active projects**: Updated within the last 2 years (exceptions for truly essential legacy tools)
- **Notable adoption**: 50+ GitHub stars OR significant industry use
- **Quality**: Well-documented, functional, and valuable to the Kafka community

### Entry format

```markdown
- [Project Name](https://github.com/org/project) - Brief description (1 sentence). Key differentiator. ![Stars](https://img.shields.io/github/stars/org/project)
```

### Pull Request Process

1. **Fork** this repository
2. **Add** your resource in the appropriate category
3. **Ensure** alphabetical ordering within categories
4. **Verify** all links work
5. **Submit** a PR with a clear description

### Categories

If your resource doesn't fit an existing category, propose a new one. Categories should have at least 3 entries.

### Quality Standards

- No promotional language or marketing speak
- Descriptions should be factual and concise
- One project per line
- No affiliate links
- Commercial tools must be clearly marked

## Updating Existing Entries

- Fix broken links
- Update star counts (automated via shields.io)
- Mark deprecated projects appropriately
- Add new features or significant updates

## Questions?

Open an issue for discussion before making large structural changes.
